Overview

This repo includes a couple tools to help easy the process of getting your enviroment setup and running. The tools are a work in progress.

If you have developed something better, we do accept pull requests :)

This documentation is aimed at demo enviroments used by Red Hat engineers

Setup

The easiest way to get started is with a linux machine (Mac support coming) and docker. You can run the following to get up an running quickly. This script will deploy and setup a docker container for development with this project.

curl -O https://raw.githubusercontent.com/redhatdod/documentation/master/setupdocker.sh
chmod +x setupdocker.sh
./setupdocker.sh

Deploying Docs to the site

The deploydocs script will automate the process of pushing everything to the production site. You need to first setup the deploydocs script for your enviroment. Copy the deploydocs script to a bin folder of your choosing, and then edit. At the top of the script there is variable that is exported. You can run the following in documents folder

sudo cp deploydocs /usr/bin
vi /usr/bin/deploydocs
export WORKINGDIR=~/therootfolderformyprojects

DO NOT EDIT THE SCRIPT IN THE REPO FOLDER!

Please copy the file to a bin, and edit it there

If you are using the docker container, everything is setup for you

Running the Dev Server

The repo includes a script to start a dev server for local development of docs. The dev server uses a nodejs package called http-server. This is a very simple, but powerful enough server for document development. The script is called devserver, and if you are not using the docker container then it will require setup.

When you first setup your container, the devserver is already running. If stop the container or reboot your machine, you will need to start the devserver from the command line. From your host machine run

sudo docker exec -it $(sudo docker ps |grep docs |awk '{print $1}') /usr/bin/devserver (1)
1 This starts the development server from your host machine

Now visit http://127.0.0.1:8080 and you can view your changes using devdocs

Using Devdocs for local development

The repo also includes a script for local development. You can use this script while you are developing a document to place into production. When combined with the devserver, you have the tools you need to get started. This script does everything that deploydocs does, except deploy the documents to production.

/usr/bin/devdocs

Destroying the docker container

There is one final script pre-bundled called destroydocker.sh

This script will destroy the docker container created by the inital setup.

From the documents repo directory, run

./destroydocker.sh
